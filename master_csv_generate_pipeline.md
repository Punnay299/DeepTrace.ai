# Deepfake & AI-Generated Video Detection System

## 1. Project Overview

This platform is a high-performance, video-only AI detection system specifically engineered to run on hardware-constrained systems—namely, laptops equipped with a single NVIDIA RTX 5060 (8GB VRAM) and running Fedora Linux. 

Rather than settling for a simple binary "fake or real" classification, this system asks the more robust question: *"What parts of this video were faked or generated, and precisely when?"* It is designed to evaluate both facial manipulation (face-swaps, fine-tuned GANs) and fully AI-generated sequences (Sora, Runway, Pika Labs).

### Hardware & Environment Constraints
*   **Target GPU**: NVIDIA RTX 5060 (8GB VRAM) limits our batch sizes and necessitates memory-efficient pipeline design.
*   **Storage**: Strict "No raw frames to disk" policy. Extracting thousands of uncompressed PNG frames would consume hundreds of gigabytes of disk space and crush I/O bandwidth.  
*   **Operating System**: Fedora Linux.
*   **Python Target**: Pyenv 3.11.8.

---

## 2. The 3-Class System

The system outputs one of 3 distinct classes, preserving indices for future audio integration (Classes 2, 3, 5, 6 are reserved):

*   **Class 0: REAL** - Entirely authentic video. No AI manipulation mathematically detected.
*   **Class 1: AI_VIDEO (Partial Swap/Injection)** - Authentic video containing injected AI segments (e.g., face-swaps localized to specific timestamps). Output generates exact start/end timestamps.
*   **Class 4: FULLY_AI_VIDEO** - Synthetic video entirely generated by AI platforms (Runway, Pika, Kling, Sora).

---

## 3. High-Level Pipeline Architecture

Running advanced video sequence models (Spatial + Temporal) usually exceeds 8GB of VRAM. To solve this, the pipeline splits responsibilities:
1.  **Stage 1-6 Data Generation Workflow**: Validates MP4s, calculates injection points mathematically, calculates bounding boxes using MTCNN, and catalogs the entire metadata map into `master.csv`. **Zero raw frames are written locally.**
2.  **Ultra-fast PyTorch DataLoader**: Reads the metadata CSV, pulls the exact 4-second video sequence dynamically via `decord`, crops to the pre-computed bounding boxes, resizes to `224x224`, and shapes the `[20, 3, 224, 224]` tensors instantly in memory.
3.  **Model 1 (Spatial)**: Fine-tuned **EfficientNet-B4** processes face-crops looking for GAN fingerprints, blending seams, and unnatural spatial artifacts. 
4.  **Model 2 (Temporal)**: A **FrameLSTM** takes encoded vectors straight from the frozen EfficientNet and scans the 20-frame sequence for unnatural timing, smooth physics, or flickering (temporal artifacts) using a fraction of the memory a TimeSformer would require.

## 4. The Data Generation Pipeline (Stages 1 through 6)

The data pipeline guarantees that the mathematical overlap between face-swap injections and video windows is absolute, generating our source-of-truth CSV files. To prepare the dataset:

### Stage 1: Validation
**Command:** `python dataset/pipeline/stage1_validate.py`
Scans `dataset/raw/class_*` folders. Under the hood, it uses `ffprobe` to verify video dimensions (>360p), valid codecs (h264, hevc, vp9), and manageable durations (30s to 300s). It skips invalid files and generates `validated_videos.csv`.

### Stage 2: Face-Swap Injection
**Command:** `python dataset/pipeline/stage2_inject.py --mock`
Class 1 (AI_VIDEO) requires precise sub-clip labeling. This stage logically injects random facial manipulation ranges into valid Class 1 videos. An 8-second face-swap might be logged from timestamp 24.3s to 32.3s. The output is `injection_log.csv`, our single source of truth for Class 1 overlap. (In production, the InsightFace ONNX swapper physically renders these.)

### Stage 3 & 4: Windowing & Labeling (The Bounding-Box Extractor)
**Command:** `python dataset/pipeline/stage3_window_and_label.py --input_csv dataset/labels/validated_videos.csv --injection_log dataset/labels/injection_log.csv --output_csv dataset/labels/windows_index.csv --device cuda`
This is where the magic happens. 
*   **Mathematical Windows**: It defines 4-second continuous windows overlapping every 2 seconds. 
*   **MTCNN Scanning**: For every window, it pulls exactly 20 frames (5 FPS). It runs MTCNN on those frames to identify the face.
*   **Normalization**: The coordinates of the face bounding box `(x1, y1, x2, y2)` are converted into normalized floats [0.0, 1.0]. This ensures dynamic resizing and completely eliminates CPU-bottlenecks during PyTorch training. If no face is found, it flags `face_detected=0`.
*   **Overlap Rule**: It cross-checks the window against `injection_log.csv`. If the window touches an injected timestamp, the frame is flagged as FAKE (`video_fake=1`). 
*   **Output**: `windows_index.csv`. **Not a single extracted frame is dumped on the disk.**

### Stage 6: CSV Matrix Generation
**Command:** `python dataset/pipeline/stage6_csv.py`
To strictly prevent data leakage between training epochs, it splits `windows_index.csv` at the **video level** (not window level) into an 80/10/10 ratio. The output yields `train.csv`, `val.csv`, and `test.csv`.

## 5. The Intelligent DataLoader (`dataset/dataloader.py`)

Usually, reading MP4s on the fly during deep learning is painfully slow, causing GPU starvation. We fixed this with **Decord** and our **Precomputed Bounding Boxes**.

*   `__getitem__` receives a row from `train.csv`.
*   It seeks instantly into the physical MP4 using `decord` and extracts the 20 sequence frames.
*   Rather than running MTCNN here (which would be catastrophically slow), it grabs the pre-computed `[x1, y1, x2, y2]` floats, un-normalizes them into pixel coordinates, clamps the boundaries safely against the video dimensions, crops the face, and performs `[224x224]` interpolations dynamically. 
*   If `face_detected=0`, it skips the crop and full-frame resizes.
*   Random horizontal flips and color-jitter augmentations are seeded identically across all 20 frames to simulate temporal consistency.
*   We benchmarked this to load sequences under **200 milliseconds**. 

You can run our integrity benchmark to prove it runs at scale: 
`python scripts/bench_dataloader.py`

## 6. AI Training Pipeline (8GB VRAM Optimized)

Both trainers are configured to utilize `FP16` mixed precision and gradient accumulation to simulate broader dataset understanding without throwing CUDA OutOfMemory errors.

### Training the Spatial Model (EfficientNet-B4)
**Command:** `python scripts/train_efficientnet.py --train_csv dataset/labels/train.csv --val_csv dataset/labels/val.csv`

The spatial model analyzes visual artifacts within single frames:
*   We simulate an effective batch size of 32 utilizing gradient accumulation steps `(4 batches * 8 acc_steps)`.
*   The frame sequence `[4, 20, 3, 224, 224]` is flattened into `[80, 3, 224, 224]`.
*   **Phase 1**: We freeze the entire backbone to isolate training securely on the classification head.
*   **Phase 2**: After early epochs, we unfreeze the final blocks with a reduced learning rate `(1e-4)`. 

### Training the Temporal Model (FrameLSTM)
**Command:** `python scripts/train_framelstm.py --train_csv dataset/labels/train.csv --val_csv dataset/labels/val.csv --backbone_weights models/efficientnet_b4_latest.pth`

The temporal model handles motion rendering failures standard image models miss:
*   It loads the fully-trained, **frozen** EfficientNet-B4. 
*   As the dataloader passes in the 20-frame sequences, the EfficientNet strips away the spatial dimensionality and passes back a `[20, 1792]` vector mapping. 
*   The raw vector passes into a Long Short-Term Memory (LSTM) network equipped with a `[256]` hidden dimension. 
*   By operating purely on vectorized space rather than 3D matrices (a trap TimeSformer systems fall into), the LSTM trains blazingly fast while capturing flickering, disjointed temporal physics, and jitter perfectly.

## 7. Getting Started

1. **Configure Environment:**
    Ensure you have `ffmpeg`. Run the following to setup Pyenv 3.11.8:
    ```bash
    pyenv local 3.11.8
    python -m venv deepfake_env
    source deepfake_env/bin/activate
    pip install --upgrade pip
    pip install -r requirements.txt
    ```

2. **Supply Raw Videos:**
    Place raw `.mp4` files into `dataset/raw/class_0_real`, `dataset/raw/class_1_ai_video`, and `dataset/raw/class_4_full_ai_video`.

3. **Orchestrate Data:**
    ```bash
    python dataset/pipeline/stage1_validate.py
    python dataset/pipeline/stage2_inject.py --mock
    python dataset/pipeline/stage3_window_and_label.py --input_csv dataset/labels/validated_videos.csv --injection_log dataset/labels/injection_log.csv --output_csv dataset/labels/windows_index.csv --device cuda
    python dataset/pipeline/stage6_csv.py
    ```

4. **Train AI:**
    ```bash
    python scripts/train_efficientnet.py --train_csv dataset/labels/train.csv --val_csv dataset/labels/val.csv
    python scripts/train_framelstm.py --train_csv dataset/labels/train.csv --val_csv dataset/labels/val.csv --backbone_weights models/efficientnet_b4_best.pth
    ```
